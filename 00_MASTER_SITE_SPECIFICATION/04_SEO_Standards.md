# Perla Toscana - SEO Standards

## Title Rules

Each page must have a unique SEO title.

Recommended format:

- Primary keyword | Perla Toscana
- Specific benefit + location | Perla Toscana

Examples:

- Casa vacanze vicino Firenze con parcheggio | Perla Toscana
- Accommodation near The Mall Firenze | Perla Toscana

Rules:

- Keep titles clear and natural.
- Avoid keyword stuffing.
- Put the main search intent early.
- Keep titles ideally under 60 characters when possible.

## Meta Description Rules

Every indexable page needs a unique meta description.

Rules:

- 140-160 characters where possible.
- Include the page value proposition.
- Include location or booking intent when relevant.
- Include a soft CTA.

Do not repeat the same description across many pages.

## H1-H6 Hierarchy

Rules:

- One H1 per page.
- H1 must match the page intent.
- H2s define major sections.
- H3s support details inside H2 sections.
- Do not skip heading levels for visual styling.
- Do not use headings only for decorative text.

Recommended page flow:

- H1: primary page promise
- H2: accommodation value / location / services / FAQs
- H3: individual service, attraction or question

## Image Naming

Use descriptive lowercase filenames.

Format:

`perla-toscana-[room-or-topic]-[descriptor].webp`

Examples:

- `perla-toscana-camera-matrimoniale.webp`
- `perla-toscana-cucina-attrezzata.webp`
- `perla-toscana-parcheggio-privato.webp`

Avoid:

- `IMG_1234.jpg`
- `photo-final-new.webp`
- keyword-stuffed filenames.

## Alt Text

Alt text must describe the image accurately.

Rules:

- Mention the room, amenity or context.
- Use keywords only when natural.
- Do not stuff locations into every alt.
- Decorative images should have empty alt text.

Example:

`Cucina attrezzata dell'appartamento Perla Toscana`

## Schema.org

Required or recommended schema:

- `LodgingBusiness`
- `LocalBusiness`
- `Accommodation` or vacation-rental-compatible schema when appropriate
- `FAQPage`
- `BreadcrumbList`
- `Organization`
- `WebSite`
- `WebPage`

Rules:

- Use only verified data.
- Do not mark up hidden FAQs.
- Use review schema only for real reviews displayed on the page and only if legally/technically appropriate.
- Validate schema before production.

## Canonical Rules

- Every indexable page must self-canonical.
- Do not canonical all languages to Italian.
- Each language page must canonical to itself.
- Tracking parameters must not create indexable duplicates.
- Anchor sections are not separate canonical pages.

## Robots

Allow crawling of all public indexable pages.

Block:

- Admin pages
- Internal preview pages
- Build assets not intended for indexing
- Search/filter URLs if they create duplicates

Do not block CSS, JS or images required for rendering.

## Sitemap

Generate XML sitemaps for indexable URLs.

Rules:

- Include only canonical URLs.
- Include localized URLs once they exist.
- Update lastmod only when content changes materially.
- Exclude anchor-only sections.

## Hreflang

Language roots:

- `it`
- `en`
- `de`
- `fr`
- `nl`

Rules:

- Use hreflang only for equivalent translated pages.
- Include self-referencing hreflang.
- Include x-default if a language selection or default global page exists.
- Do not point hreflang to missing pages.

## Internal Linking Rules

Every page must link to:

- The relevant parent page.
- At least one commercial landing page.
- Book Direct where appropriate.
- Related services or attractions.

Money pages must receive links from:

- Homepage
- Navigation or footer
- Relevant guides
- FAQ blocks
- Related attraction pages

## EEAT Guidelines

Demonstrate experience and trust through:

- Real property photos.
- Exact amenities.
- Transparent policies.
- CIN/CIR.
- Real reviews.
- Google Business Profile consistency.
- Local knowledge based on the property's actual location.
- Clear owner/contact information.

Do not invent distances, awards, partnerships, reviews or amenities.

## Public Transport SEO Rules

Public transport is a competitive advantage and should support the Florence, Location and Arrival clusters.

Rules:

- Use the verified bus stop name: SR69 Deposito Gas.
- Use verified stop IDs: FIV074 and FIV075.
- State the walking distance as approximately 200 metres.
- Explain the route to Florence through Figline Valdarno railway station and Florence Santa Maria Novella.
- Do not publish fixed bus timetables.
- Link to Autolinee Toscane for current schedules: <https://www.at-bus.it/>
- Use this topic to support, not replace, the private parking positioning.

## Guest Information Hub SEO Rules

The Guest Information Hub supports user trust, long-tail SEO and direct booking conversion.

Rules:

- Each guest-info page must have a unique practical intent.
- Each page must link back to Location, FAQ, Book Direct and Arrival Guide where relevant.
- Guest-info pages must not compete with money pages. They support commercial landing pages through internal links.
- Do not publish fixed third-party prices, timetables, opening hours or availability unless the data is reliable and maintainable.
- For taxi, transfer, restaurant, wine-tour and private-driver pages, instruct guests to contact providers directly.
- Use official provider links where possible.
