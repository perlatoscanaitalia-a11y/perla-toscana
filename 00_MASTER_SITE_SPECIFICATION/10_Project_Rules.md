# Perla Toscana - Project Rules

These rules are permanent standards for every future page, component and feature.

## Strategic Rules

- Direct booking is the primary business objective.
- SEO must support booking quality, not just traffic.
- Every page must have a clear role in the topical authority map.
- Do not create thin pages.
- Merge overlapping topics whenever possible.
- Do not invent amenities, reviews, distances or policies.
- Do not publish fixed public transport timetables; link to the official provider for current schedules.
- Do not publish fixed taxi, private transfer, wine-tour, restaurant, pharmacy, petrol-station or EV-charging data unless it can be maintained.

## Architecture Rules

- Use approved language roots: `/it/`, `/en/`, `/de/`, `/fr/`, `/nl/`.
- Initially implement only Italian and English unless requested otherwise.
- Do not publish placeholder translated pages.
- Use logical URL groups: services, guides, places, location, rules and information.
- Every page must be reachable through navigation, footer, breadcrumbs or contextual links.

## Design Rules

- Follow the brand palette exactly.
- Use Cormorant Garamond for headings and Inter for body text.
- Keep the design elegant, minimal and fast.
- Use real images.
- Do not promote the garden until ready.
- Do not use oversized decorative sections that distract from booking.
- Cards must not be nested inside cards.

## Content Rules

- Every page needs one primary intent.
- Every page needs one primary keyword or a distinct long-tail variant.
- Every authority article must link to at least one money page.
- Every money page must include a booking CTA.
- FAQs must answer real traveler questions.
- CTAs must be clear and repeated at natural decision points.

## SEO Rules

- Unique title and meta description for every indexable page.
- One H1 per page.
- Self-canonical every indexable page.
- Use hreflang only for real translated equivalents.
- Add structured data only when accurate.
- Use internal links deliberately.
- Do not keyword stuff.
- Public transport pages and sections must support the Florence, Location and Arrival clusters.
- Guest Information pages must support trust and booking confidence, not distract from the main commercial pages.

## Performance Rules

- Optimize images before publishing.
- Use WebP.
- Lazy-load below-the-fold media.
- Keep JavaScript minimal.
- Protect Core Web Vitals.
- Test mobile layout before delivery.

## Trust Rules

- Display CIN/CIR where required.
- Show real reviews only.
- Keep rules clear: no pets, no smoking, no parties.
- Make WhatsApp/contact easy to find.
- Make booking terms transparent.
- Present public transport accurately: SR69 Deposito Gas, stop IDs FIV074/FIV075, approximately 200 m walk, bus to Figline Valdarno railway station, train to Florence Santa Maria Novella.
- For taxi companies, private transfers, restaurants, wine tours and private drivers, visitors must be encouraged to contact providers directly for current prices and availability.
- Food and drink recommendations must use Google Maps links and must not hardcode opening hours, prices or phone numbers.
- 1950 American Diner - FIGLINE V.NO may be recommended for lunch and dinner, especially for families and guests arriving late, with wide parking and a 1950s atmosphere.

## Development Rules

- Reuse components.
- Keep styles consistent.
- Do not hardcode duplicate metadata when a structured metadata system exists.
- Validate pages for mobile, SEO, accessibility and performance.
- Do not ship broken links or empty language pages.

## Pre-Publish Checklist

Before any page goes live:

- Intent is clear.
- Primary keyword is unique.
- URL is correct.
- Title and meta description are unique.
- H1 is correct.
- Images are optimized.
- Alt text is accurate.
- Internal links are present.
- CTA is present.
- FAQ is present if useful.
- Schema is valid if used.
- Mobile view is clean.
- No unsupported claims.
